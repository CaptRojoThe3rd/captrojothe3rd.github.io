Adding CHR data is required* if you want your NES game to.. well.. display anything on the screen. A base NROM cartridge can store 8 KiB of CHR-ROM.

To include CHR data, you must first create the file that the CHR-ROM is stored in. Name it 'alpha.chr', and make it 8 KiB in size using a hex editor. You should put it in a folder called 'chr' for organization, since YYCHR will create other files depending on what you do with it.

*Well, there is a way to render stuff without CHR data, but it's extremely limited. It's similar to the Atari 2600's graphics, but even more limited. You see, if you disable background and sprite rendering in the PPU, the PPU still displays a solid color. Normally, this is just the background color, stored at $3f00 in the PPU's memory. However, if you set the PPU's bus address (Register $2006) to a different palette address between $3f01-$3f1f, the PPU will actually display that color instead of the background color at $3f00.
