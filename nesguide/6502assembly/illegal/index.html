Illegal instructions are instructions that were never intended to exist on the 6502, but do because of bugs or other reasons. Some are perfectly stable and could have been included in the documentation of the original processor, while others are unstable or highly unstable and generally shouldn't be used. The followiing instructions are the ones supported by CA65.


ALR - ANDs accumulator with value, then shifts right. Equivalent to an AND instruction followed by an LSR instruction.


alr #$ee



ANC - ANDs accumulator with value, then copies bit 7 into carry flag.


anc #$ee



ARR - ANDs accumulator with value, then rotates right. Equivalent to an AND instruction followed by a ROR instruction.


arr #$ee



AXS - Sets X to X AND Accumulator minus a value.


axs #$ee



DCP - Decrements value at memory address, sets flags as if value has been compared with the Accumulator. Equivalent to DEC instruction followed by a CMP instruction.


dcp $ee
dcp $ee,x
dcp $eeee
dcp $eeee,x
dcp $eeee,y
dcp ($ee,x)
dcp ($ee),y



ISC - Increments value at memory address, subtracts value at memory address from Accumulator. Equivalent to INC instruction followed by a SBC instruction.


isc $ee
isc $ee,x
isc $eeee
isc $eeee,x
isc $eeee,y
isc ($ee,x)
isc ($ee),y



LAS - Sets Accumulator, X, and Stack Pointer to Stack Pointer AND value at memory address.


las $eeee,y



LAX - Loads Accumulator and X with value at memory address. Equivalent to an LDA instruction followed by a TAX instruction. Note that while the immediate value mode of this instruction does exist, it is unstable and involves a "magic" constant value. Yes, the bug has a bug.


lax $ee
lax $ee,y
lax $eeee
lax $eeee,y
lax ($ee,x)
lax ($ee),y



RLA - Rotates value at memory address left, then ANDs Accumulator with value at memory address. Equivalent to a ROL instruction followed by an AND instruction.


rla $ee
rla $ee,x
rla $eeee
rla $eeee,x
rla $eeee,y
rla ($ee,x)
rla ($ee),y



RRA - Rotates value at memory address right, then adds value at memory address with the Accumulator. Equivalent to a ROR instruction followed by an ADC instruction.


rra $ee
rra $ee,x
rra $eeee
rra $eeee,x
rra $eeee,y
rra ($ee,x)
rra ($ee),y



SAX - Stores Accumulator AND X and stores it at memory address.


sax $ee
sax $ee,y
sax $eeee
sax ($ee,x)



SLO - Shifts value at memory address left, then ORs Accumulator with value at memory address. Equivalent to an ASL instruction followed by an ORA instruction.


slo $ee
slo $ee,x
slo $eeee
slo $eeee,x
slo $eeee,y
slo ($ee,x)
slo ($ee),y



SRE - Shifts value at memory address right, then XORs Accumulator with value at memory address. Equivalent to an LSR instruction followed by an EOR instruction.


sre $ee
sre $ee,x
sre $eeee
sre $eeee,x
sre $eeee,y
sre ($ee,x)
sre ($ee),y



STP - Stops execution. This isn't actually supported by CA65, but it is somtimes useful for debugging. Since it isn't supported by CA65, you need to write the opcode manually or use a macro.


stp

; Since CA65 doesn't support this instruction in NMOS 6502 mode, a macro like this can be used
.macro stp

.byte $22

.endmacro
